# Filing preferences — template (you own this)

This is the **human-owned** rule layer the `file-butler` skill anchors to. The
product ships this *template*; your filled-in copy lives in **your** space and
is treated as **your data** — a data reset clears it, the shipped template does
not. The butler reads it before proposing anything and suggests additions after
each job; **you** edit it, it obeys it. Auto-generating these rules would be
net-negative: they are your taste, which no scan can infer.

Keep it small and true; grow it as corrections happen.
